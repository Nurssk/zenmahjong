# Zen Mahjong
**Zen Mahjong** — современная веб-платформа для игры в Mahjong, созданная как стартап-прототип для коротких умных игровых сессий, отдыха, фокуса и соревновательного режима.
Проект объединяет классический Mahjong Solitaire, систему прогресса, обучение, аналитику игрока, лидерборды, магазин, персонажей-наставников и турнирный режим с ежедневной общей раскладкой для всех пользователей.
---
## Основная идея проекта
Zen Mahjong — это не просто игра, а полноценная игровая платформа, где пользователь может:
- пройти обучение;
- играть в разные уровни сложности;
- сохранять прогресс;
- продолжать незавершённую игру;
- смотреть историю своих партий;
- анализировать личную статистику;
- участвовать в ежедневных турнирах;
- соревноваться в лидерборде;
- использовать подсказки, отмену хода и shuffle;
- покупать игровые ресурсы;
- выбирать тему интерфейса;
- взаимодействовать с сенсеем-наставником.

## Авторизация

### Возможности авторизации

## Онбординг и обучение
В игре есть отдельный обучающий режим для новых пользователей.
### Что есть в обучении
- пошаговое объяснение правил Mahjong;
- обучающий персонаж;
- объяснение простыми словами;
- демонстрация базовых игровых механик;

Обучение помогает пользователю быстро понять правила перед началом основной игры.

## Игровой процесс
В Zen Mahjong реализована полноценная игровая механика Mahjong Solitaire.
### Основные правила
Игрок может убрать плитку только если:
- плитка свободна;
- плитка не заблокирована другими плитками;
- выбрана пара одинаковых плиток;
- обе плитки доступны для удаления.
Заблокированные плитки удалить нельзя.
---
## Режимы сложности
В игре есть три уровня сложности.
### Лёгкий уровень
- доступны подсказки;
- доступна отмена хода;
- подсвечиваются возможные доступные ходы;
- подходит для новичков.
### Средний уровень
- плитки отображаются одинаково;
- игрок сам ищет доступные пары;
- подсказки и отмена хода доступны ограниченно.
### Сложный уровень
- нет подсказок;
- нет отмены хода;
- игрок должен пройти раскладку полностью самостоятельно;
- подходит для опытных игроков.
---
## Сохранение игры
Игра сохраняется в базе данных Firebase.
Пользователь может:
- создать новую игру;
- продолжить прошлую незавершённую игру;
- вернуться к игровому процессу после выхода;
- сохранить результат после завершения партии.
Если игра была начата, но не завершена, пользователь может продолжить её позже.
---
## История игр
Все завершённые и незавершённые игры сохраняются.
В истории учитываются:
- дата игры;
- сложность;
- результат;
- время прохождения;
- количество очков;
- количество использованных подсказок;
- количество отмен хода;
- статус игры: победа, поражение или незавершённая партия.
История используется для аналитики профиля и рейтингов.

## Аналитика профиля
В профиле пользователя есть отдельная вкладка с аналитикой.
Система анализирует большое количество данных по играм:
- общее количество игр;
- количество побед;
- количество поражений;
- количество незавершённых игр;
- процент побед;

Аналитика помогает пользователю видеть свой прогресс и улучшать стратегию.

## Очки и прогресс
В игре есть система очков.
Очки зависят от:
- скорости прохождения;
- сложности уровня;
- количества правильных ходов;
- использования подсказок;
- использования отмены хода;
- результата игры.
Результаты сохраняются и используются в leaderboard.

## Подсказки, Undo и Shuffle
В игре реализованы дополнительные игровые инструменты.
### Подсказка
Помогает найти доступную пару плиток.
### Undo Move
Позволяет отменить последний ход.
### Shuffle
Перемешивает оставшиеся плитки, если игрок оказался в сложной ситуации.
Количество подсказок и undo ограничено. Дополнительные подсказки и отмены можно получить через внутриигровой магазин за монеты или самоцветы.
---
## Завершение игры
Игра может завершиться двумя способами.
### Победа
Если пользователь убрал все плитки, игра завершается победой.
После победы:
- сохраняется результат;
- начисляются очки;
- обновляется статистика;
- результат попадает в историю игр;
- данные могут участвовать в рейтинге.
### Поражение
Если доступных ходов больше нет, а плитки ещё остались, игра завершается поражением.
После поражения:
- результат сохраняется;
- игра помечается как проигранная;
- обновляется аналитика профиля.

## Сенсей
В игре есть сенсей — персонаж-наставник, который помогает игроку во время партии.
### Что делает сенсей
- даёт игровые советы;
- помогает понять стратегию;
- подсказывает, какие ходы могут быть полезными;
- предупреждает о потенциально плохих ходах;
- создаёт ощущение AI Coach.
На текущем этапе сенсей работает на rule-based логике, но интерфейс сделан как у полноценного AI-наставника.
---
## Скины сенсея
В игре есть система персонажей-сенсеев.
### Доступные сенсеи
#### Айро
Базовый сенсей, доступный по умолчанию.
#### Угвей
Дополнительный сенсей, которого можно купить за самоцветы в магазине.
У каждого сенсея свой визуальный стиль и характер. При покупке нового сенсея меняется его скин в интерфейсе.

В магазине можно получить или купить:
- монеты;
- самоцветы;
- дополнительные подсказки;
- undo;

## Турнирный режим
В Zen Mahjong есть отдельный режим турнира.
### Как работает турнир
- каждый день создаётся одна одинаковая раскладка для всех пользователей;
- все игроки проходят одинаковый уровень;
- результаты сохраняются в Firebase;
- игроки соревнуются по очкам и времени;
- данные отображаются в leaderboard.
Турнирный режим делает игру соревновательной и повышает ценность ежедневного возвращения пользователя.
---
## Leaderboard
В проекте есть система рейтингов.
### Виды рейтинга
- глобальный рейтинг;
- рейтинг по стране;
- рейтинг по городу;
- рейтинг ежедневного турнира;
- общий рейтинг по всем турнирам.
В leaderboard отображаются:
- имя игрока;
- очки;
- время прохождения;
- количество игр;
- среднее время;
- позиция игрока;
- данные по текущему турниру;
- данные по всем турнирам.
На текущем этапе leaderboard частично заполнен тестовыми fake-данными, так как реальных пользователей пока мало. Это сделано для демонстрации работы интерфейса и Firebase-структуры.
---
## Daily Challenge
Daily Challenge — ежедневная общая раскладка для всех игроков.
Система сохраняет:
- дату турнира;
- раскладку;
- результат игрока;
- очки;
- время прохождения;
- пользователя;
- город пользователя, если он указан.
Это позволяет строить ежедневные соревнования и сравнивать результаты игроков.
